Timer

每一个 java.util.Timer 对象对应一个后台线程 TimerThread 和任务队列 TaskQueue，支持执行一次任务调度或周期性任务调度

TimerThread 不断循环从 TaskQueue 中取出最近要执行的任务，通过当前系统时间和需要调度时间判断如果要调度任务，那么执行调度，否则 wait 相应的时间

private void mainLoop() {  
    while (true) {  
        try {  
            TimerTask task;  
            boolean taskFired;  
            synchronized(queue) {  
                // Wait for queue to become non-empty  
                while (queue.isEmpty() && newTasksMayBeScheduled)  
                    queue.wait();  
                if (queue.isEmpty())  
                    break; // Queue is empty and will forever remain; die  
  
                // Queue nonempty; look at first evt and do the right thing                long currentTime, executionTime;  
                task = queue.getMin();  
                synchronized(task.lock) {  
                    if (task.state == TimerTask.CANCELLED) {  
                        queue.removeMin();  
                        continue;  // No action required, poll queue again  
                    }  
                    currentTime = System.currentTimeMillis();  
                    executionTime = task.nextExecutionTime;  
                    if (taskFired = (executionTime<=currentTime)) {  
                        if (task.period == 0) { // Non-repeating, remove  
                            queue.removeMin();  
                            task.state = TimerTask.EXECUTED;  
                        } else { // Repeating task, reschedule  
                            queue.rescheduleMin(  
                              task.period<0 ? currentTime   - task.period  
                                            : executionTime + task.period);  
                        }  
                    }  
                }  
                if (!taskFired) // Task hasn't yet fired; wait  
                    queue.wait(executionTime - currentTime);  
            }  
            if (taskFired)  // Task fired; run it, holding no locks  
                task.run();  
        } catch(InterruptedException e) {  
        }  
    }  
}

TaskQueue 是通过最小堆实现的，每当调度任务被放进队列里，根据每个任务下次调度的时间排序，把最先调度的任务放在堆顶。每次取到最先调度的任务时判断是否周期任务，如果是的话修改任务下次调度的时间，重新排序；否则把任务出队列

Timer 的实现简单易懂，比较轻量级。它的问题也比较明显，调度是由一个后台单线程实现的，当某个任务需要很长执行周期的时候，会阻塞下个任务被调度；并且在使用上也要求任务类必须继承 TimerTask，不够灵活。因此在 Timer 的文档上也推荐使用 Java 5.0 引入的 ScheduledThreadPoolExecutor

ScheduledThreadPoolExecutor

ScheduledThreadPoolExecutor 最明显的优势在于调度任务的是线程池，而非单线程，通过这种线程隔离的机制就能降低下个任务调度被阻塞的风险

ScheduledThreadPoolExecutor 继承自 ThreadPoolExecutor，它同样实现了一个最小堆实现的优先级队列，任务按到期时间排序，任务未到时间时，取任务的线程会阻塞等待。对于周期执行的任务，出队后会计算下次执行时间再放回队列

while (poolRunning) {
    task = delayQueue.take(); // 阻塞直到最近的任务到时间
    if (task.isPeriodic()) {
        runTask(task);
        task.setNextTime();
        delayQueue.put(task);
    } else {
        runTask(task);
    }
}

但即使是线程池调度，里面的线程一旦被某个任务占满，就无法去执行队列里的下一个任务。比如你的定时任务执行了很长时间，线程就被这个任务一直占用，因为 DelayedWorkQueue 是按下次执行时间排序的，不会自动跳过“到时间但还没执行的任务”去执行后面的任务。所以如果有长任务，更好的方式是移到独立线程池执行并监控

TimeWheel

时间轮 是一种 实现延迟功能（定时器） 的巧妙算法。如果一个系统存在大量的任务调度，时间轮可以高效的利用线程资源来进行批量化调度。把大批量的调度任务全部都绑定在时间轮上，通过时间轮进行所有任务的管理，触发以及运行。能够高效地管理各种延时任务，周期任务，通知任务等

时间轮（TimingWheel）算法应用范围非常广泛，我们熟悉的 Linux Crontab，以及常用的框架/中间件如 Dubbo、Netty、Akka、Quartz、ZooKeeper 、Kafka 等，几乎所有和 时间任务调度 都采用了时间轮的思想

有了线程池定时调度，为什么还需要时间轮？因为时间轮更适合海量定时任务，有以下几个核心差异

任务数量规模的差异

  • 线程池调度：
    任务按时间排序，堆越大，插入/删除开销就越大（O(log n)），百万任务时性能下降明显

  • 时间轮调度：
    任务分桶放在环形数组里，绝大部分操作是 O(1)，非常适合需要管理成千上万甚至百万个延时任务的场景（比如网络连接超时、心跳检测）

精度与延迟特性

  • 线程池调度：

    • 精度高，可以做到毫秒/纳秒级
    • 适合精确调度的定时任务，比如报表生成、周期计算

  • 时间轮调度：

    • 精度取决于 tick（比如 100 ms 一格），可能会有抖动
    • 适合对精度要求不高的场景（网络超时、缓存过期等）

线程占用

  • 线程池调度：

    • 需要线程去等待到期任务（DelayedWorkQueue.take() 会阻塞），核心线程数过小会有阻塞风险

  • 时间轮调度：

    • 一个线程就能扫描桶并分发任务，不会因为任务数量暴增而增加线程数，资源占用极低
